堆和优先队列

发表于 2021-03-23 更新于 2021-03-28 分类于 算法与数据结构

优先队列

普通队列：先进先出；后进后出

优先队列：出对顺序和如对顺序；和优先级有关

为什么使用优先队列？

计算机世界中，最典型的例子就是操作系统中进行任务的调度。在现代操作系统中，会同时执行多个任务。操作系统要为这多个任务分配计算资源，包括分配 CPU 的时间片。具体去分配这些资源时，操作系统要看各个任务的优先级去动态选择优先级最高的任务执行。

堆

二叉堆

二叉堆就是满足一些特殊性质的二叉树。

  • 二叉堆是一棵完全二叉树
  • 堆中某个节点的值总是不大于其父节点的值（最大堆），相应的可以定义最小堆

向堆中添加元素和 Sift Up

如下图所示，我们要向这一棵二叉树添加一个值为 52 的节点。虽然此时，依然满足完全二叉树的性质，但是并不满足任意一个节点都小于等于它的父亲节点这个性质。

此时，应该从 52 开始依次与它的父亲节点作比较。52 的父亲节点是 16，16 小于 52，打破了堆的性质，所以 52 与 16 交换位置，此时，以 52 为根的二叉树再次满足了堆的性质。

还需要继续看，对于 52 来说，它现在在索引为 4 的位置，在这个新的位置上，与它的父亲节点作比较。可以看出，52 依然大于它的父亲节点，再次进行位置的交换。

下面继续看，52 小于等于它的父亲节点，就不需要进行交换。

取出堆中最大的元素和 Sift Down

首先，对于堆来说，之所以叫它最大堆，就是因为我们从堆中取出元素，只取出堆顶的元素，这个元素是堆中存储元素中最大的元素。

如图所示，我们将 62 这个元素取出后，相当于要将左右两棵子树融合成一个堆，相对来说比较复杂。

使用一个小技巧：将堆中最后一个元素，顶到堆顶去。

这样操作后，数组 0 的位置就是16，不过数组最后一个元素的位置也是16，我们把数组最后一个元素删除掉，

这样首先从元素的个数上成功减少了一个元素，并且减少的元素就是原来堆顶的元素。

但是目前，堆顶的元素打破了堆的性质。需要对堆顶元素进行 Sift Down（下沉）操作。

每次要下沉位置的元素，都去和它的两个孩子作比较，选择两个孩子中更大的元素，如果这个更大的元素比它自己还要大，就进行元素交换。

如图所示，16 与 52 作比较， 52 比 16 大，交换位置。

继续进行比较，交换

现在 16 来到了索引为 4 的位置，16 比 15 要大，此时下沉操作就结束了。

基于堆的优先队列

Leetcode 上优先队列相关的问题

优先队列的经典问题： top K 问题

在 N 个元素中选出最大的 K 个元素

在 N 个元素中选出最小的 K 个元素

思路：使用优先队列，维护当前看到的最小的 K 个元素。对于每一个新的数据，如果比这 K 个最小元素中最大的还小，则替换。应该使用最大堆而不是使用最小堆，因为我们要找到这 K 个最小元素中的最大值。

Java 标准库中的优先队列

LeetCode 215. 数组中的第 K 个最大元素

剑指 Offer 40. 最小的 k 个数

使用快排思想和优先队列的比较

快排：时间：O(n) ; 空间：O(1)

优先队列：时间：O(nlogk)；空间：O(k)

优先队列的优势：不需要一次性知道所有数据。适合数据流、极大规模数据。
