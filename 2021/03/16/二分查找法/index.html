二分查找法的变种： upper

有时，我们不一定要查找 target ，而是要查找大于 target 的最小值。搜索范围 arr[l, r]。其中，

r = arr.length。因为要找的值可能比 l 所在的位置的元素还要小，所以最小的索引是 l 。但是如

果要找的值比 r 所在索引的元素还要大，所有最大的范围可以是 arr.length。

如下图所示，这组数据可能是一个班级学生在一场考试中的成绩，老师想知道的不是某个分数是否有学生得到，而是想知道比如大于 60 的最小值。在及格了的同学中得到最差的分是多少分。

搜索范围 [l, r]，对于 r ，不应该指向最右侧的元素，而应该指向最右侧元素加一的位置。如果 target 比 23 还要小的话，很显然，23 就是答案。但是，如果 target 是99，或者比 99 还要大的话，数组中没有这样一个元素，就可以使用 arr.length 表示位置。

对于二分查找法，中间的元素就指向 72，我们要根据中间的这个元素来缩减我们搜索的范围。我们需要修改 r 的值，但不是 r = mid - 1 ，而应该是 r = mid。因为 72 比 60 大，所有比 60 大的元素都有可能是我们要找的元素。

继续找，mid 变成了 65，又是比 60 大，所以继续 r = mid。

继续找，mid 变成了 56，这种情况下，我们要找的是大于 target 的最小值，56 比 60 小，所以不可能是我们要找的，所以 l = mid + 1。

至此，l 和 r 汇合到一起。65 就是要找的大于 60 的最小值。

二分查找法的变种： ceil

给定一个含有重复元素的有序数组，如 [1, 1, 3, ,3 ,5 ,5, 7, 7]。比如查找的是5，如果数组中存在元素，返回最大索引，如下图。

如果查找6，数组中并不包含6，返回 upper。

可以先求 target 的 upper ，然后 upper - 1 ，如果 upper - 1 的索引元素等于 target ，则 upper - 1就是 ceil 值，如果 upper - 1 的索引元素不等于 target ，则 upper 就是 ceil 值。

二分查找法的变种： lower

如下图，寻找小于 target 的最大值。搜索范围 arr[l, r] ，l = -1。

假设我们现在要找小于 85 的最大值。mid 位置的值为 69，比 85 小，然后在 mid 索引的右边继续寻找。l = mid。如果 arr[mid] >= target r = mid - 1。

但是会有问题，如在 [1, 1, 3, 3, 5, 5] 中寻找 2 的lower，会陷入死循环。会停在 l = 0, r = 1。

当 l = 0 , r = 1 时，mid = l + (r - l) / 2 = 0 + (1 - 0) / 2 = 0。arr[0] < 2，所以 l = 0。会进入死循环。

当 l 与 r 相邻时，根据 mid 的计算方法，mid 一定等于 l，如果 arr[mid] < target，l = mid，l 的值就会一直不变。

解决方案： m = l + (r - l + 1) / 2
