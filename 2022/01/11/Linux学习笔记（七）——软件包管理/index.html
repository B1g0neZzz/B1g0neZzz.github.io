软件包管理

RPM 命令管理

RPM 包命名规则

httpd-2.2.15-15.e16.centos.i686.rpm

httpd 软件包名

2.2.15 软件版本

15 软件发布的次数

e16.centos 适合的 Linux 平台

i686 适合的硬件平台

rpm rpm 包扩展名

RPM 包依赖性

树形依赖：a => b => c

环形依赖：a => b => c => a

模块依赖：模块依赖查询网站：

​ www.rpmfind.net

RPM 安装

rpm -ivh 包全名

选项：

-i install 安装

-v verbose 显示详细信息

-h hash 显示进度

--nodeps 不检测依赖性

RPM 包升级

rpm -Uvh 包全名

RPM 卸载

rpm -e 包名

选项：

-e erase 卸载

RPM 命令查询

查询是否安装

rpm -q 包名 查询包是否安装

rpm -qa 查询所有已经安装的 RPM 包

rpm -qi 查看包的详细信息

查询包中文件安装位置

rpm -ql 包名

查询系统文件属于哪个 RPM 包

rpm -qf 系统文件名 查询系统文件属于哪个软件包（file）

查询包的依赖性

rpm -qR 包名

选项：

-R 查询软件包的依赖性

-p 查询未安装包信息

RPM 校验和文件提取

rpm -V 已安装的包名 verify

RPM 包中文件提取

比如

rpm -qf /bin/ls 查询 ls 命令属于哪个软件包

mv /bin/ls /tmp/ 误删除 ls 命令

rpm2cpio /mnt/cdrom/Packages/coreutils-8.4-19.e16.i686.rpm | cpio -div ./bin/ls 提取 RPM 包中 ls 命令到当前目录的 /bin/ls 下

cp /root/bin/ls /bin/ 把 ls 命令复制回 /bin/ 目录，修复文件损失。

RPM 包管理——yum 在线管理

常用 yum 命令

查询

yum list 查询所有可用软件包列表

yum search 关键字 搜索服务器上所有和关键字相关的包

安装

yum -y install 包名

-y 自动回答 yes

升级

yum -y update 包名

卸载

yum -y remove 包名

非常危险，不安全，慎用

光盘 yum 源搭建

第一步，挂载

mount /dev/sr0 /mnt/cdrom/

第二步，让网络 yum 源文件失效，可以更改这几个文件的名称

第三步，修改光盘 yum 源文件

源码包

与 RPM 包的区别

默认安装位置

RPM 一般安装在默认位置：

源码包安装在指定位置当中，一般是 /usr/local/软件名/

默认安装位置不同带来的影响

RPM 包安装的服务可以使用系统服务管理命令（service）来管理，例如 RPM 包安装的 apache 的启动方法是：

/etc/rc.d/init.d/httpd start

service httpd start

而源码包安装的服务则不能被服务管理命令管理，因为没有安装到默认路径中。所以只能用绝对路径进行服务的管理，如：

/usr/local/apache2/bin/apachectl start

源码包安装过程

源代码保存位置：usr/local/src/

软件安装位置：/usr/local/

第一步：软件配置与检查

./configure --prefix=/usr/local/apache2

第二步：编译

make

第三步：编译安装

make install

脚本安装包

人为把安装过程写成了自动安装的脚本。

sudo 权限

root 把本来只能超级用户执行的命令赋予普通用户执行。

sudo 的操作对象是系统命令。

sudo 使用

visudo 实际修改的是 /etc/sudoers 文件

root ALL = (ALL) ALL

用户名 被管理主机的地址 = （可使用的身份）授权命令（绝对路径）

举例:

visudo

sc ALL = /sbin/shutdown -r now 授权 sc 用户可以重启服务器

普通用户执行：

sudo -l 查看可用的 sudo 命令

sudo /sbin/shutdown -r now 普通用户执行 sudo 赋予的命令
